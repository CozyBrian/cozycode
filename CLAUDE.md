## Codebase Index

This project has a living `docs/` folder with architecture, implementation, patterns, decisions, and changelog files.

### Session Start
- Read `docs/architecture.md` and `docs/implementation.md` before doing any work.
- These files contain the project map; do not re-scan the codebase from scratch.

### Red Flags
- Check the indexed docs before searching.
- Use targeted searches only for information not covered by the docs.
- Update the relevant docs after every feature or bugfix.
- Tell subagents to read `docs/architecture.md` and `docs/implementation.md` first.

### After Changes
1. Inspect changed files and direct neighbors only.
2. Update architecture, implementation, or patterns when those areas changed.
3. Add an architectural decision only when a meaningful tradeoff was made.
4. Append a dated entry to `docs/changelog.md`.
