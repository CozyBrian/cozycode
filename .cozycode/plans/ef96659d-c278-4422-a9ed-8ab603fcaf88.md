# Plan: Slash Command Autocomplete / Suggestions

## Goal
When the user types `/` in the Prompt textarea, show a dropdown of available slash commands. As the user continues typing, filter the suggestions. Allow accepting a suggestion with Tab or by typing the full command.

## Current State
- **Slash commands** are handled in `apps/tui/src/app.tsx` `submit()` (lines ~180-205): exact-match on `/new`, `/clear`, `/model`, `/models`, `/plan`, `/build`, `/help`, `/quit`, `/exit`.
- **CommandPalette** (`apps/tui/src/components/CommandPalette.tsx`) has a separate list of commands with descriptions for the Ctrl+P overlay.
- **Prompt** (`apps/tui/src/components/Prompt.tsx`) renders an `@opentui/react` `<textarea>` with `ref`, `onSubmit`, `onKeyDown`, `keyBindings`.
- The textarea's `TextareaRenderable` ref exposes `.plainText` (getter) and `.setText()` (setter).

## Design Decisions

### Approach: Dropdown suggestions above the prompt
- Render a suggestion popup box inside the Prompt component when the input starts with `/`.
- Filter commands in real-time as the user types.
- Tab accepts the top suggestion (or the only one if filtered to one).
- Escape dismisses the suggestion popup.
- The popup disappears when input no longer starts with `/`.

### Shared command definitions
Extract the slash command list into a shared constant (likely in a new file `apps/tui/src/commands.ts`) so both `SlashSuggestions` and `app.tsx`'s `submit()` use the same source of truth.

## Files to Create

### 1. `apps/tui/src/commands.ts` (NEW)
Define slash command metadata in one place:

```ts
export interface SlashCommand {
  command: string;    // e.g. "/new"
  name: string;       // e.g. "New chat"
  description: string; // e.g. "Start a fresh session"
}

export const SLASH_COMMANDS: SlashCommand[] = [
  { command: "/new", name: "New chat", description: "Start a fresh session" },
  { command: "/clear", name: "Clear", description: "Clear the transcript" },
  { command: "/model", name: "Switch model", description: "Choose a different model" },
  { command: "/plan", name: "Plan mode", description: "Read-only research; no edits" },
  { command: "/build", name: "Build mode", description: "Execute changes (default)" },
  { command: "/help", name: "Help", description: "Show keybindings" },
  { command: "/quit", name: "Quit", description: "Exit cozycode" },
];
```

Also export a helper:
```ts
export function matchSlashCommands(query: string): SlashCommand[] {
  if (!query.startsWith("/")) return [];
  return SLASH_COMMANDS.filter(c => c.command.startsWith(query));
}
```

### 2. `apps/tui/src/components/SlashSuggestions.tsx` (NEW)
A small popup component that renders filtered slash command suggestions:

```tsx
// Props:
// - query: string (the current input text starting with /)
// - selectedIndex: number
// Returns matching commands filtered by query
// Renders a compact box with commands, highlighting the selected one
```

Design:
- Renders as a `box` positioned above the prompt (using flexDirection column, rendered before the textarea).
- Shows up to all matching commands with their descriptions.
- Highlights the selected index with `selectedBackgroundColor`.
- If query is empty or doesn't start with `/`, renders nothing.

## Files to Modify

### 3. `apps/tui/src/components/Prompt.tsx`
Add suggestion state and rendering:

**New state:**
- `suggestionQuery`: string | null — the current `/` filter text, null when not showing suggestions
- `selectedIndex`: number — which suggestion is highlighted (0-based)

**New logic in `onKeyDown`:**
- After any key that modifies content (printable chars, backspace, delete):
  - Use `queueMicrotask(() => { const text = input.current?.plainText ?? ""; ... })` to read the updated text
  - If `text` starts with `/`, set `suggestionQuery = text` and compute `selectedIndex = 0`
  - Otherwise, set `suggestionQuery = null`
- On Tab:
  - If `suggestionQuery` is set and there are matches:
    - `key.preventDefault()`
    - Replace textarea content with the full command from the top suggestion
    - Clear suggestions
  - If Tab and no suggestions: fall through to existing toggle-mode behavior
- On Escape:
  - If suggestions are visible: clear suggestions, `key.preventDefault()`
  - Otherwise: let the key propagate

**New rendering:**
- Conditionally render `<SlashSuggestions>` above the textarea when `suggestionQuery` is not null.

**New prop:** None needed (suggestions are self-contained in Prompt).

### 4. `apps/tui/src/app.tsx`
- Import `SLASH_COMMANDS` from `../commands.ts` and `matchSlashCommands`.
- Refactor the `submit()` function's slash-command switch to use the shared command list instead of hardcoded strings. Specifically, use `SLASH_COMMANDS.find(c => c.command === text)` and dispatch accordingly. The `/models` alias and `/exit` alias can be handled as additional lookups.
- Keep the `default:` case for unknown commands (shows "Unknown command: ...").

### 5. `apps/tui/src/components/CommandPalette.tsx` (optional)
- Could optionally import from `commands.ts` to reuse descriptions, but the current OPTIONS list is fine as-is. Low priority.

## Detailed Implementation Steps

1. **Create `apps/tui/src/commands.ts`** with `SlashCommand`, `SLASH_COMMANDS`, `matchSlashCommands`.

2. **Create `apps/tui/src/components/SlashSuggestions.tsx`**:
   - Import `matchSlashCommands` and `theme`.
   - Takes `query: string` and `selectedIndex: number`.
   - Returns `null` if no matches.
   - Renders a compact bordered box with the filtered commands.
   - Each command shows the command name highlighted (e.g., `/new`) and description in muted.

3. **Modify `apps/tui/src/components/Prompt.tsx`**:
   - Add `useState` for `suggestionQuery` and `selectedIndex`.
   - Import `matchSlashCommands` and `SlashSuggestions`.
   - In the textarea's `onKeyDown`:
     ```tsx
     onKeyDown={(key) => {
       // Existing tab handling for mode toggle:
       if (key.name === "tab") {
         if (suggestionQuery) {
           const matches = matchSlashCommands(suggestionQuery);
           if (matches.length > 0) {
             key.preventDefault();
             input.current?.setText(matches[selectedIndex].command);
             setSuggestionQuery(null);
             setSelectedIndex(0);
             return;
           }
         }
         key.preventDefault();
         onToggleMode();
         return;
       }
       
       if (key.name === "escape") {
         if (suggestionQuery) {
           key.preventDefault();
           setSuggestionQuery(null);
           setSelectedIndex(0);
           return;
         }
       }
       
       // Schedule update after textarea processes the key
       queueMicrotask(() => {
         const text = input.current?.plainText ?? "";
         if (text.startsWith("/") && text.length >= 1) {
           setSuggestionQuery(text);
           setSelectedIndex(0);
         } else {
           setSuggestionQuery(null);
           setSelectedIndex(0);
         }
       });
     }}
     ```
   - Render `<SlashSuggestions query={suggestionQuery} selectedIndex={selectedIndex} />` above the textarea, inside the prompt box.

4. **Modify `apps/tui/src/app.tsx`**:
   - Import `SLASH_COMMANDS` from `../commands.ts`.
   - Refactor `submit()` slash handling to be driven by `SLASH_COMMANDS`:
     ```tsx
     if (text.startsWith("/")) {
       // Handle aliases
       const effective = text === "/models" ? "/model" : text === "/exit" ? "/quit" : text;
       const cmd = SLASH_COMMANDS.find(c => c.command === effective);
       if (!cmd) {
         // unknown command
         setHistory(h => [...h, { id: `unknown-command:${Date.now()}`, kind: "error", text: `Unknown command: ${text}` }]);
       } else {
         dispatchSlashCommand(cmd.command);
       }
       setInputKey(k => k + 1);
       return;
     }
     ```
   - Extract `dispatchSlashCommand` function or reuse the existing switch (keeping it works too).

5. **Test manually:**
   - Type `/` → should show all 7 commands.
   - Type `/p` → should show only `/plan`.
   - Type `/b` → should show only `/build`.
   - Press Tab → textarea fills with the full command.
   - Press Escape → suggestions disappear.
   - Type `/unknown` → no suggestions (empty list, popup hides).
   - Type non-slash text → no suggestions.
   - Existing Tab-to-toggle-mode still works when not in slash context.

## Risks & Alternatives

### Risk: `queueMicrotask` timing
The `onKeyDown` fires synchronously before the textarea processes the key. Using `queueMicrotask` should give us the updated text, but there's a theoretical risk that in some edge cases the text isn't updated yet. Alternative: maintain our own text buffer by tracking keystrokes, but that's more complex and error-prone.

### Risk: `@opentui/react` textarea API limitations
If `setText()` doesn't exist or behaves differently than expected, we may need to use a different mechanism to accept suggestions. The `TextareaRenderable` type is imported from `@opentui/core` — we should verify `.setText()` exists. If not, we can use the `inputKey` trick (incrementing it resets the textarea with new content via `defaultValue` or similar).

### Alternative: Inline ghost-text autocomplete
Instead of a dropdown, we could show the first suggestion as dimmed/grey text inline (like fish shell or zsh autosuggest). This is more complex to implement in a TUI textarea and would require the textarea to support placeholder/ghost text.

### Alternative: Use CommandPalette overlay
Instead of a new dropdown, we could open the existing CommandPalette when `/` is typed. Downside: it wouldn't filter as you type, and the UX would be different from what users expect from slash-command suggestions.
